Web Workers Demo: The Basics

1.Advantages: run expensive operations in isolated thread without interrupting main thread

deal with normal expensive data

image processing

lazy loading

canvas paint and render

MVVM mode, data-driven development, deal with big data in front end

2.Disadvantages: Worker thread can not access the DOM or the parent page, Worker thread can only communicate through postMessage()

Same origin limitation

4.Basic Steps

  1. You create a web worker(two types: Worker and SharedWorker).
  2. You tell the web worker what to do. (For example, crunch these numbers!)
  3. You start the web worker.
  4. When the web worker is done, it tells you, and your code goes from there. (For example, show the results on your page.)

5.How to stop
